# 2026-07-13 작업 기록

## Front 디자인 재설계

기존 전시 포스터형 디자인을 제거하고, 전쟁 역사 기록물을 도서관 열람실에서 검색하고 읽는 구조로 다시 설계했다.

## 변경한 파일

- `front/web/app/page.tsx`
- `front/web/app/globals.css`

## 변경 이유

- 화면이 단순 전시물처럼 보이는 문제를 줄이고, 사용자가 기록물을 직접 검색하고 열람하는 경험을 강화했다.
- 본문 카드나 상세 버튼을 찾아야만 정보가 열리는 느낌을 줄이고, 검색 결과 선택과 열람대 갱신이 바로 연결되도록 화면 구조를 정리했다.
- 데스크톱과 모바일에서 글자 크기, 줄바꿈, 버튼 배치가 깨지지 않도록 반응형 레이아웃을 보강했다.

## 구현 내용

- 메인 화면을 `REFERENCE LIBRARY`, `PUBLIC READING DESK`, `WA-REF 001` 중심의 기록물 열람실 컨셉으로 교체했다.
- 히어로 영역을 "사료를 찾고, 맥락으로 읽습니다." 메시지 중심으로 재구성했다.
- 분류 서가 영역을 전쟁과 전투, 인물과 증언, 지도와 장소, 조약과 외교 네 개의 입구로 정리했다.
- 검색 영역을 자료실 작업대 형태로 재설계했다.
- 열람 화면, 사서 노트, 리딩 가이드, 북마크, 인용 복사, 탭 전환, 글자 크기 조절 기능이 한 화면 안에서 자연스럽게 보이도록 스타일을 전면 교체했다.
- 모바일에서 긴 제목과 버튼이 화면 밖으로 밀리지 않도록 줄바꿈, 최소 너비, 버튼 폭, 제목 크기 규칙을 보강했다.

## 검증 결과

- 요구사항 검증: 기존 디자인을 유지하지 않고 새로운 도서관 기록물 열람실 디자인으로 교체했다.
- 정방향 검증: 로컬 개발 서버에서 메인 페이지 HTTP 200 응답을 확인했다.
- 역방향 검증: 모바일 런타임에서 `innerWidth 492`, 최대 문서 폭 `477`로 가로 오버플로가 없음을 확인했다.
- 화면 검증: Edge headless로 데스크톱과 모바일 화면을 캡처해 주요 영역의 겹침 여부를 확인했다.

## 남은 위험 요소

- 현재 `front/` 전체가 git 기준 untracked 상태라 변경 파일 단위의 git diff 비교는 제한적이다.
- 공개 기록 데이터가 아직 적기 때문에, 실제 기록물이 늘어난 뒤 카드 밀도와 필터 사용성을 다시 확인해야 한다.

# 2026-07-12 작업 기록

## Front 초기 구성 및 게시 준비

전쟁 역사 아카이브의 공개 프론트엔드 구조를 처음 구성하고, GitHub Pages로 정적 배포할 수 있는 기반을 만들었다.

## 변경한 주요 영역

- `front/web`: Next.js 기반 공개 아카이브 사이트
- `front/mobile`: React Native 모바일 앱 골격
- `front/.github/workflows/pages.yml`: GitHub Pages 배포 워크플로
- `front/web/content/archive.json`: 공개 기록 데이터
- `front/web/app/archive/[id]/page.tsx`: 기록 상세 페이지
- `front/web/app/sitemap.ts`, `front/web/app/robots.ts`, `front/web/app/manifest.ts`: 검색 노출과 정적 사이트 메타 파일

## 구현 내용

- `front`를 독립 프론트엔드 저장소로 구성했다.
- 공개 사이트를 Next.js 앱으로 만들고, 정적 빌드와 GitHub Pages 배포가 가능하도록 설정했다.
- 전쟁 역사 기록 목록과 상세 기록 페이지를 구성했다.
- `archive.json`을 기준으로 공개 기록 데이터를 읽는 구조를 만들었다.
- sitemap, robots.txt, manifest, JSON-LD 기반 메타데이터를 구성했다.
- React Native 모바일 앱의 기본 골격을 추가했다.
- GitHub Actions 기반 Pages 배포 워크플로를 추가했다.
